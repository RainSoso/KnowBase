# 概述及基本原则

## 1. 目的

本文档旨在规范前后端接口设计、开发和协作流程，确保接口的一致性、可维护性和易用性。

## 2. 适用范围

适用于公司所有前后端接口的设计、开发和维护工作。

## 3. 基本原则

* 前端应只关注<font color="#c00000">渲染逻辑</font>，而不应该关注数据的处理逻辑。
* 接口返回的数据应能<font color="#c00000">直接展示在界面</font>上。
* 一个功能应<font color="#c00000">避免多个接口嵌套</font>调用获取数据，后台应该处理好一次性返回。
*  <font color="#c00000">响应格式应是JSON</font>，避免多层嵌套JSON的出现。
* 实时刷新数据应使用WebSocket等机制，而非前端轮询。

> [!TIP]
> 
> 根据这个原则，<font color="#c00000">后端返回的数据应该是所见即所得的数据</font>，而不应该是一层包一层的复杂的JSON，需要前端额外来处理很复杂的去重，排序、深浅拷贝，变形等处理。
> <div>前端在渲染每一个div，每一个table的cell时，不用担心数据结构会频繁更改从而导致页面崩溃。</div>
